Unit 8: Interfaces and Abstract Classes

Interfaces

An interface in Java is a collection of abstract methods. It defines a contract that implementing classes must adhere to. Interfaces support multiple inheritance, allowing a class to implement multiple interfaces.

// Example of an interface
interface Shape {
void draw();
double calculateArea();
}
// Example of a class implementing an interface
class Circle implements Shape {
void draw() {
System.out.println("Drawing a circle");
}
double calculateArea() {
// Implementation of area calculation for a circle
return 3.14 * radius * radius;
}
}

Abstract Classes

An abstract class in Java is a class that cannot be instantiated on its own and may contain abstract methods. Abstract methods are declared without implementation and must be implemented by subclasses.

// Example of an abstract class
abstract class Shape {
abstract void draw();
abstract double calculateArea();
}
// Example of a subclass extending an abstract class
class Circle extends Shape {
void draw() {
System.out.println("Drawing a circle");
}
double calculateArea() {
// Implementation of area calculation for a circle
return 3.14 * radius * radius;
}
}
